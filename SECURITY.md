# Security Policy

## Reporting a vulnerability

Please do not open a public issue for security-sensitive reports.

Instead, report vulnerabilities privately to:

- `[security contact to be added]`

Include:

- a description of the issue
- reproduction details if available
- impact assessment if known

## Scope

Use private reporting for vulnerabilities involving authentication, session handling, ESI token storage, database exposure, or other sensitive runtime behavior.

Non-sensitive bugs and general defects can still be reported through normal GitHub issues.
